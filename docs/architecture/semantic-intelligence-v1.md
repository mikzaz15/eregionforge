# Semantic Intelligence v1

## Purpose

Semantic Intelligence v1 is the first quality-upgrade pass on top of the durable canon layer.

The goal is not full semantic reasoning yet.
The goal is to make the existing compiled research surfaces more investment-native, more explainable, and more trustworthy.

## What Improved

### Contradictions

The contradiction layer now looks for:

- same-theme directional disagreement, not just shallow token overlap
- newer-versus-older claim tension on the same topic
- pricing, demand, margin, guidance, customer, regulatory, product, and timing disagreement patterns
- timeline tension that reflects thesis impact instead of simple date mismatch

Each contradiction still carries rationale and supporting references.

### Catalysts

Catalyst compilation now behaves more like investment research:

- earnings checkpoints are separated from generic dated events
- product ramps, regulatory milestones, pricing inflections, customer signals, and macro shifts are identified more directly
- catalyst descriptions now explain why the event matters, how it affects the thesis, and any linked contradiction risk
- confidence now reflects support density, contradiction burden, and date precision rather than only source presence

### Freshness Attribution

Stale alerts now point to more specific causes:

- relevant changed source themes
- contradiction posture shifts
- catalyst posture shifts
- support-density changes
- source-diversity changes
- exact-date timeline changes

This keeps stale alerts closer to knowledge quality and thesis impact.

### Confidence

Confidence is still heuristic, but it is more defensible.

Current confidence models now use combinations of:

- support density
- source diversity
- contradiction burden
- freshness burden
- exact-date precision where relevant
- section coverage for dossier-style outputs

## What Remains Heuristic

This upgrade still does not do:

- entity resolution
- semantic embeddings or vector ranking
- fragment-level contradiction proofs
- true causal attribution for thesis changes
- probabilistic confidence calibration

The system is still using transparent heuristic compilation.

## Next Semantic Upgrades

Good next targets:

1. entity-aware comparison across claims, catalysts, and timeline events
2. support density at the fragment and evidence-link level rather than only object counts
3. stronger semantic freshness attribution tied to changed assumptions, not only changed objects
4. contradiction clustering so multiple tensions on one theme resolve into one cleaner integrity object
5. better catalyst invalidation logic once user review state becomes first-class
